PARTE A: Ingenieria de Requerimientos e Historias de Usuario

1. Requerimientos Funcionales (RF)

Los siguientes requerimientos funcionales constituyen el nucleo operativo del sistema Clipboard TTS Monitor. Cada requerimiento se identifica con la nomenclatura estricta RF-NN y se vincula con las historias de usuario correspondientes.

Requerimientos Funcionales del Sistema
IDRequerimientoModuloPrioridad
RF-01 Monitoreo de Portapapeles mediante Listener Pasivo. El sistema debe registrar el listener nativo AddClipboardFormatListener sobre una ventana oculta de mensajes para recibir notificaciones WM_CLIPBOARDUPDATE cuando el contenido del portapapeles cambia, sin interferir ni bloquear el flujo nativo de CTRL+C de la aplicacion de origen. Clipboard Listener Alta
RF-02 Extraccion Asincrona de Texto con Anti-Deadlock Backoff. Tras detectar un cambio en el portapapeles, el sistema debe encolar una tarea en un hilo trabajador (Worker Thread) con un retardo controlado de ~50ms antes de invocar OpenClipboard, GetClipboardData(CF_UNICODETEXT) y CloseClipboard, para evitar bloqueos mutuos con la aplicacion de origen. Adicionalmente, el sistema debe filtrar el texto, ignorando silenciosamente aquellos que contengan 40 caracteres o menos. Clipboard Listener Alta
RF-03 Sintesis de Voz Text-to-Speech (TTS) Asincrona. El texto extraido del portapapeles debe ser procesado mediante el motor SAPI5 (ISpVoice) utilizando la bandera SPF_ASYNC, permitiendo que la reproduccion se ejecute sin congelar el hilo principal de procesamiento de mensajes Win32 ni bloquear la escucha de atajos globales. TTS Engine Alta
RF-04 Control de Salto de Reproducción (CTRL+1). El sistema debe registrar un hotkey global CTRL+1 que detenga la reproducción actual (saltar). Si hay elementos en la cola, el sistema continuará automáticamente con la reproducción del siguiente texto. Hotkey Manager Alta
RF-05 Repeticion del Ultimo Registro Hablado (CTRL+3). El sistema debe mantener un bufer circular en memoria (historico LIFO de capacidad configurable, minimo 1 entrada) con el ultimo texto sintetizado exitosamente. Al activar CTRL+3, debe reproducir el contenido de dicho bufer, independientemente del estado actual del portapapeles. TTS Engine / Historico Alta
RF-06 Detención Total y Vaciado de Cola (ALT+1). El sistema debe registrar un hotkey global ALT+1 que detenga completamente todas las reproducciones actuales y vacíe la cola de textos. El programa debe quedar en estado de escucha esperando nuevos textos. Hotkey Manager / State Machine Alta
RF-07 Control Dinamico de Velocidad de Reproduccion (20 Niveles). El sistema debe exponer un control de velocidad con 20 niveles discretos, mapeados linealmente desde $0.25x$ (nivel 1) hasta $2.5x$ (nivel 20), correspondientes al rango numerico de SAPI de $-10$ a $+10$. La conversion debe seguir la formula: $Rate_{SAPI} = round( (velocidad - 1) \times \frac{20}{19} - 10 )$. TTS Engine Media
RF-08 Interfaz de Bandeja del Sistema (System Tray) con Menu Contextual. La aplicacion carece de ventana principal persistente. Debe exponer un icono en la bandeja del sistema mediante Shell_NotifyIcon(NIM_ADD) con un menu contextual (TrackPopupMenu) que incluya: lista de voces disponibles, control deslizante de velocidad, opcion de reconfiguracion de atajos y opcion de salida. Win32 UI / Tray Icon Alta
RF-09 Enumeracion y Seleccion de Voces TTS Instaladas. El menu contextual debe enumerar todas las voces SAPI5 disponibles en el sistema mediante SpEnumTokens(SPCAT_VOICES, NULL, NULL, &pEnum), permitiendo al usuario seleccionar la voz activa en tiempo de ejecucion mediante ISpVoice::SetVoice(). TTS Engine / Win32 UI Media
RF-10 Persistencia de Configuracion mediante config.ini. El sistema debe leer y escribir un archivo estructurado config.ini (formato INI estandar con secciones [Hotkeys], [TTS], [Audio], [General]) que almacene: codigos de teclas virtuales (VK) y modificadores para cada atajo, voz seleccionada, nivel de velocidad, ruta de salida de grabaciones, y estado de habilitacion del modulo de grabacion. Config Persistence Media
RF-11 Reconfiguracion Dinamica de Atajos de Teclado. El sistema debe permitir, a traves del menu contextual, la modificacion de las combinaciones de teclas para cada funcion (copiar-escuchar, detener, repetir, toggle hook) persistiendo los nuevos valores VK y modificadores en config.ini y re-registrando los hotkeys mediante UnregisterHotKey / RegisterHotKey sin requerir reinicio de la aplicacion. Hotkey Manager / Config Media
RF-12 Modulo de Grabacion de Audio (Desactivado por Defecto). El sistema debe incluir un modulo de grabacion que, cuando este habilitado (configuracion RecordingEnabled=0 por defecto), persista cada texto sintetizado como archivo de audio comprimido en formato AAC dentro de contenedor M4A (o WMA), utilizando Windows Media Foundation (IMFSinkWriter o topologia de codificacion ASF). Audio Logger Baja
RF-13 Nomenclatura ISO 8601 para Archivos de Grabacion. Los archivos de audio generados por el modulo de grabacion deben nombrarse estrictamente bajo la norma YYYYMMDD-HHMMSS_TTS.m4a (ej. 20260628-143052_TTS.m4a), utilizando la hora local del sistema en el momento de la sintesis. Audio Logger Baja
RF-14 Instalador MSI Corporativo (WiX Toolset v4). El proyecto debe incluir un instalador MSI generado con WiX Toolset v4 que: instale los binarios en ProgramFiles6432Folder segun la arquitectura, cree la entrada de auto-arranque en HKCU\Software\Microsoft\Windows\CurrentVersion\Run, y proteja el archivo config.ini contra sobreescritura en reinstalaciones (componente con NeverOverwrite="yes"). Deployment Media
RF-15 Gestión de Cola de Reproducción. El sistema debe implementar una cola de reproducción para gestionar múltiples textos copiados secuencialmente. Mientras un texto se reproduce, los nuevos textos (mayores a 40 caracteres) se encolarán hasta un máximo de N elementos (configurable, por defecto 1). TTS Engine Alta
RF-16 Pausa y Reanudación (CTRL+2). El sistema debe registrar un hotkey global CTRL+2 que permita pausar la reproducción actual. Al presionarlo nuevamente, debe reanudar la reproducción desde el mismo punto. Hotkey Manager Alta

2. Requerimientos No Funcionales (RNF)

Los requerimientos no funcionales mitigan riesgos documentados en la literatura tecnica de Windows y garantizan la calidad del sistema.

Requerimientos No Funcionales del Sistema
IDRequerimientoCategoriaPrioridad
RNF-01 No Interferencia del Portapapeles. El sistema no debe bloquear, modificar ni interferir con el flujo nativo de CTRL+C. El diseno debe basarse exclusivamente en listeners pasivos (AddClipboardFormatListener) con un retardo asincrono controlado de ~50ms en un hilo secundario antes de invocar OpenClipboard, evitando bloqueos mutuos (deadlocks) con la aplicacion de origen. Concurrencia / Seguridad Critica
RNF-02 Bucle de Mensajes No Bloqueante (UI Thread Integrity). El motor TTS debe configurarse en modo asincrono nativo (SPF_ASYNC) de SAPI5 o ejecutarse en un hilo trabajador independiente (Worker Thread) para evitar que la reproduccion de textos extensos congele el procesamiento de los hotkeys globales (WM_HOTKEY) en el hilo principal de Win32. El tiempo de respuesta de los hotkeys no debe exceder 50ms bajo carga maxima. Concurrencia / Rendimiento Critica
RNF-03 Arquitectura de Tipos Seguros (x86/x64). Todo el codigo fuente debe utilizar tipos polimorficos de precision de puntero: INT_PTR, LONG_PTR, WPARAM, LPARAM, UINT_PTR, DWORD_PTR. Queda estrictamente prohibido el uso de tipos fijos de 32 bits (DWORD, UINT, int) para almacenar valores de puntero o handles, garantizando la compilacion cruzada limpia para arquitecturas x86 (32-bit) y x64 (64-bit). Portabilidad Critica
RNF-04 Gestion de Memoria Segura (RAII & COM). Todo recurso COM debe gestionarse mediante punteros inteligentes: CComPtr<T> (ATL) o winrt::com_ptr<T> (C++/WinRT). Los handles de Windows (HWND, HMENU, HGLOBAL, etc.) deben liberarse mediante RAII custom (auto_handle, unique_ptr con deleter). Queda prohibido el uso de new/delete crudo para objetos COM; CoCreateInstance debe ir seguido inmediatamente de asignacion a CComPtr. Seguridad / Estabilidad Critica
RNF-05 Soporte Unicode Estricto. Toda API interna y externa procesara cadenas de caracteres anchos exclusivamente: wchar_t, std::wstring, LPCWSTR, LPWSTR. El proyecto debe compilarse con UNICODE y _UNICODE definidos. El formato de clipboard a utilizar es CF_UNICODETEXT (UTF-16LE). La configuracion INI debe codificarse en UTF-16LE con BOM o UTF-8 con BOM para soporte completo de eñes, tildes y diéresis del español. Internacionalizacion Critica
RNF-06 Manejo Robusto de Errores (Graceful Degradation). Toda funcion que invoque una API de Windows debe verificar el valor de retorno HRESULT / BOOL / HANDLE. En caso de error, el sistema debe: registrar el error en el log de depuracion (OutputDebugString), notificar al usuario mediante un tooltip transitorio del icono de bandeja (NOTIFYICONDATA con NIF_INFO), y continuar operando sin terminar el proceso. Fiabilidad Alta
RNF-07 Bajo Consumo de Recursos. La aplicacion en estado idle debe consumir menos de 10MB de memoria RAM y menos del 0.1% de CPU. El uso de polling esta prohibido; solo se permiten mecanismos basados en eventos (listeners, callbacks, mensajes de Windows). Eficiencia Alta
RNF-08 Compilacion Hibrida x86/x64. El proyecto Visual Studio debe configurarse con dos plataformas de compilacion: Win32 (x86) y x64. Las rutas de salida deben separarse: $(SolutionDir)bin\$(Platform)\$(Configuration)\. Los componentes COM deben registrarse segun la arquitectura del sistema destino (regsvr32 de 32-bit para x86, de 64-bit para x64). Portabilidad / Despliegue Alta
RNF-09 Seguridad del Instalador. El instalador MSI debe: instalar sin privilegios de administrador (per-user install, Package Scope="perUser"), firmar digitalmente los binarios y el paquete MSI, proteger config.ini con el atributo NeverOverwrite="yes" para preservar la configuracion del usuario en actualizaciones, y registrar el auto-arranque en HKCU (no HKLM). Seguridad Media
RNF-10 Minima Interferencia Visual. La aplicacion no debe crear ninguna ventana principal visible. La unica interfaz de usuario permitida es el icono de la bandeja del sistema y sus menues contextuales. No se permiten ventanas de dialogo modales excepto para la reconfiguracion de atajos. Usabilidad Media

3. Historias de Usuario y Criterios de Aceptacion BDD

Historia de Usuario 1: Monitoreo de Portapapeles y Sintesis de Voz

HU-01: "Como usuario con discapacidad visual que trabaja con documentos digitales, quiero que el sistema detecte automaticamente cuando copio texto con CTRL+C y lo reproduzca en voz alta, para poder escuchar el contenido sin necesidad de pegarlo en otra aplicacion."
Criterio de Aceptacion 1.1 - Deteccion y Sintesis Basica

Dado que la aplicacion esta en estado Escuchando y el portapapeles del sistema contiene texto plano previamente copiado
Cuando el usuario presiona CTRL+C en cualquier aplicacion y el sistema recibe WM_CLIPBOARDUPDATE
Entonces el sistema extrae el texto del portapapeles en un hilo trabajador tras un retardo de 50ms, lo envia al motor SAPI5 con SPF_ASYNC, y la reproduccion de voz comienza dentro de los siguientes 500ms

Criterio de Aceptacion 1.2 - No Interferencia con la Aplicacion Origen

Dado que el usuario esta trabajando en Microsoft Word con un documento grande abierto
Cuando el usuario copia texto repetidamente (mas de 50 veces en 10 segundos)
Entonces Word no experimenta bloqueos, congelamientos ni perdida de datos, y el portapapeles opera con flujo nativo ininterrumpido; el sistema procesa cada copia secuencialmente sin invocar OpenClipboard desde el hilo principal

Historia de Usuario 2: Control Global de Detencion

HU-02: "Como usuario que escucha textos largos sintetizados, quiero poder detener la reproduccion de voz instantaneamente con un atajo de teclado global, para interrumpir la lectura cuando ya haya obtenido la informacion que necesitaba."
Criterio de Aceptacion 2.1 - Detencion Inmediata

Dado que el sistema esta reproduciendo un texto de 5000 caracteres mediante SAPI5 en modo asincrono
Cuando el usuario presiona CTRL+1 desde cualquier aplicacion (incluso cuando Clipboard TTS Monitor no tiene el foco)
Entonces la reproduccion de voz se detiene completamente dentro de los 50ms siguientes, el motor TTS queda en estado listo, y el sistema notifica mediante tooltip: "Reproduccion detenida"

Historia de Usuario 3: Repeticion del Ultimo Texto Hablado

HU-03: "Como usuario que necesita reescuchar informacion importante, quiero poder repetir el ultimo texto que fue sintetizado sin tener que volver a copiarlo del documento original, para ahorrar tiempo y esfuerzo."
Criterio de Aceptacion 3.1 - Repeticion desde Buffer Historico

Dado que el sistema ha sintetizado exitosamente el texto "El informe financiero muestra un crecimiento del 15%" y desde entonces el portapapeles ha cambiado a otro contenido diferente
Cuando el usuario presiona CTRL+3
Entonces el sistema reproduce exactamente el texto "El informe financiero muestra un crecimiento del 15%" desde el bufer historico interno, ignorando el contenido actual del portapapeles, y muestra tooltip: "Reproduciendo ultimo texto"

Historia de Usuario 4: Control de Estado del Hook

HU-04: "Como usuario que necesita desactivar temporalmente el monitoreo para realizar copias privadas sin sintesis de voz, quiero poder activar y desactivar la escucha del portapapeles con un atajo, para mantener control sobre cuando el sistema opera."
Criterio de Aceptacion 4.1 - Toggle de Estado

Dado que la aplicacion esta en estado Escuchando (icono de bandeja en verde)
Cuando el usuario presiona ALT+1
Entonces el sistema transiciona al estado Inactivo, cambia el icono de bandeja a gris, descarta todos los eventos WM_CLIPBOARDUPDATE subsiguientes sin procesarlos, y muestra tooltip: "Monitoreo desactivado"

Criterio de Aceptacion 4.2 - Reactivacion

Dado que la aplicacion esta en estado Inactivo
Cuando el usuario presiona ALT+1 nuevamente
Entonces el sistema transiciona al estado Escuchando, restaura el icono verde, y comienza a procesar nuevamente los eventos de portapapeles

Historia de Usuario 5: Control de Velocidad de Reproduccion

HU-05: "Como usuario que consume grandes volumenes de texto, quiero ajustar la velocidad de reproduccion de voz a 20 niveles distintos, desde muy lento hasta muy rapido, para adaptar la lectura a mi velocidad de comprension personal."
Criterio de Aceptacion 5.1 - Mapeo Lineal de Velocidades

Dado que el usuario abre el menu contextual del icono de bandeja y selecciona "Velocidad"
Cuando el usuario selecciona el nivel 10 (velocidad media)
Entonces el sistema calcula $Rate_{SAPI} = round((10-1) \times \frac{20}{19} - 10) = 0$, configura ISpVoice::SetRate(0), persiste SpeedLevel=10 en config.ini, y todas las reproducciones subsecuentes utilizan velocidad $1.0x$ (normal)

Criterio de Aceptacion 5.2 - Persistencia entre Sesiones

Dado que el usuario configuro la velocidad en nivel 16 y cerro la aplicacion
Cuando el usuario reinicia la aplicacion
Entonces el sistema lee SpeedLevel=16 de config.ini, calcula $Rate_{SAPI} = +6$, y aplica automaticamente la velocidad correspondiente a $1.76x$

Historia de Usuario 6: Interfaz de Bandeja del Sistema

2. Vista Logica

La vista logica describe la descomposicion modular del software en cinco modulos funcionales principales, cada uno con responsabilidad unica y interfaces bien definidas. La comunicacion entre modulos se realiza exclusivamente mediante interfaces programaticas; no se permite acceso directo a datos internos.

2.1 Descripcion de Modulos

Modulo 1: TrayIconModule (Win32 UI)

Responsabilidad: Gestionar el icono de la bandeja del sistema, sus tooltips, y el menu contextual (TrackPopupMenu). No contiene logica de negocio; actua como puente entre el usuario y los modulos core.
AspectoDefinicion
API PrincipalShell_NotifyIcon, TrackPopupMenu, LoadIconMetric, DestroyMenu
Estructura ClaveNOTIFYICONDATA con NIF_ICON | NIF_MESSAGE | NIF_TIP | NIF_GUID
Mensaje CustomWM_TRAYICON = WM_USER + 1 (maneja WM_LBUTTONUP, WM_RBUTTONUP)
Registro ExploradorRegisterWindowMessage(L"TaskbarCreated") para recrear icono tras crash de explorer.exe
DependenciasHotkeyManager, TTSEngine, ConfigManager, AudioLogger (para queries de estado)

Modulo 2: ClipboardListener

Responsabilidad: Recibir notificaciones de cambio del portapapeles mediante listener pasivo, extraer el texto de forma asincrona con retardo anti-deadlock, y entregarlo al TTSEngine. Nunca bloquea el portapapeles de forma persistente.
AspectoDefinicion
API PrincipalAddClipboardFormatListener, RemoveClipboardFormatListener, OpenClipboard, GetClipboardData(CF_UNICODETEXT), CloseClipboard
Mensaje WindowsWM_CLIPBOARDUPDATE
Hilo de EjecucionWorker Thread con CreateThread / std::thread; el retardo de 50ms se aplica via Sleep(50) o WaitForSingleObject con timer
Mutex de Proteccionstd::mutex para serializar accesos concurrentes al portapapeles; timeout de 500ms para evitar bloqueo indefinido
DependenciasTTSEngine (entrega texto), StateMachine (consulta si esta Escuchando)

Modulo 3: TTSEngine

Responsabilidad: Sintetizar texto a voz mediante SAPI5 (ISpVoice), gestionar la seleccion de voces, controlar la velocidad de reproduccion, y notificar al HistoryBuffer cuando una sintesis se completa. Opera exclusivamente en modo asincrono. // Ejecutado en Worker Thread
void ClipboardListener::ExtractTextAsync() {
    // PASO 1: Retardo anti-deadlock
    Sleep(50);  // Permite que la app origen libere el clipboard
    
    // PASO 2: Intentar abrir clipboard con timeout
    for (int attempt = 0; attempt < 5; ++attempt) {
        if (OpenClipboard(NULL)) {
            break;
        }
        Sleep(5);  // Backoff incremental
    }
    
    // PASO 3: Verificar formato disponible
    if (!IsClipboardFormatAvailable(CF_UNICODETEXT)) {
        CloseClipboard();
        return;
    }
    
    // PASO 4: Extraer datos
    HANDLE hData = GetClipboardData(CF_UNICODETEXT);
    if (hData == NULL) { CloseClipboard(); return; }
    
    LPCWSTR pszText = static_cast<LPCWSTR>(GlobalLock(hData));
    if (pszText == NULL) { CloseClipboard(); return; }
    
    // PASO 5: Copiar a buffer propio (wchar_t / std::wstring)
    std::wstring text(pszText);
    
    GlobalUnlock(hData);
    CloseClipboard();
    
    // PASO 6: Entregar al hilo principal via PostMessage
    PostMessage(g_hMainWnd, WM_APP_CLIPBOARD_TEXT, 0, 
                reinterpret_cast<LPARAM>(new std::wstring(text)));
}

    // level in [1, 20] -> rate in [-10, +10]
    // Mapeo lineal: rate = round((level - 1) * 20/19 - 10)

Anexo B: Enumeracion de Voces SAPI5

El siguiente fragmento conceptual ilustra el patron de enumeracion de voces instaladas mediante la API SAPI5. Este patron se implementa en el modulo VoiceEnumerator y el resultado se expone al usuario a traves del menu contextual del icono de bandeja.

Patron de Enumeracion de Voces SAPI5
// ============================================================
// Enumerar todas las voces TTS instaladas en el sistema
// utilizando SpEnumTokens helper y CComPtr para gestion COM
// ============================================================
HRESULT EnumerateInstalledVoices(
    std::vector<std::pair<std::wstring, std::wstring>>& outVoices
) {
    // outVoices: vector de pares <TokenID, DisplayName>
    CComPtr<IEnumSpObjectTokens> spEnum;
    
    // Enumerar todas las voces en la categoria SPCAT_VOICES
    HRESULT hr = SpEnumTokens(
        SPCAT_VOICES,    // L"HKEY_LOCAL_MACHINE\\SOFTWARE\\Microsoft\\Speech\\Voices"
        NULL,            // Sin atributos requeridos
        NULL,            // Sin atributos opcionales
        &spEnum
    );
    if (FAILED(hr)) return hr;
    
    // Iterar sobre los tokens encontrados
    while (hr == S_OK) {
        CComPtr<ISpObjectToken> spToken;
        ULONG celtFetched = 0;
        hr = spEnum->Next(1, &spToken, &celtFetched);
        if (FAILED(hr) || celtFetched == 0) break;
        
        // Obtener el Token ID completo (para SetVoice y persistencia)
        LPWSTR pszTokenId = NULL;
        if (SUCCEEDED(spToken->GetId(&pszTokenId))) {
            std::wstring tokenId(pszTokenId);
            CoTaskMemFree(pszTokenId);
            
            // Obtener el nombre descriptivo de la voz
            LPWSTR pszName = NULL;
            if (SUCCEEDED(spToken->GetStringValue(L"Name", &pszName))) {
                std::wstring name(pszName);
                CoTaskMemFree(pszName);
                
                // Opcional: obtener idioma y genero
                LPWSTR pszLang = NULL;
                if (SUCCEEDED(spToken->GetStringValue(L"Language", &pszLang))) {
                    // pszLang contiene el LANGID (ej. "409" para en-US)
                    CoTaskMemFree(pszLang);
                }
                
                outVoices.emplace_back(tokenId, name);
            }
        }
        // spToken se libera automaticamente al salir del scope (CComPtr)
    }
    
    return S_OK;
}
